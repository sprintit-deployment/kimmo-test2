APPLICATION HIGHLIGHTS [back]

By this module will helps you to maintain supermodules/module packages.
Sometimes we have a lot of modules available in the module, and at the initial stage of the server, we need to install all/almost of them. So one by one install all of them, it's a time-consuming process.
To avoid such a situation, this module will be helpful. This module provides several processes like Install/Upgrade/Remove packages of modules.

 

FUNCTIONALITY [back]

FEATURES

  • Install all the modules available in the package.
  • Update all the modules available in the package.
  • Update newly added modules to the package.
  • Remove module package which are removed from specific path.
  • Dependecy Graph

Configuration [back]

No need for configuration.

Modules Packages [back]

1. Go to-> Apps-> Module Packages.
2. On Cliking menu "Module Packages",
3. You'll see all the avaialble packages with their path(s).

Packages Operations [back]

1. Once you open the any package record, you'll see the list of all the modules which are available in that package.

There are four buttons are available for several operations as below:
2. Install All: To install all the 'Not Installed' modules from a specific package.
3. Upgrade All: To upgrade all the 'Installed' modules from a specific package.
4. Reset App List: To update the new modules to be added on a specific package.
5. Hard Reset App List: To remove the package/modules, if the module folder(s) have been removed from the server.

Dependency Graph [back]

1. Dependency Graph: The second tab available on the package view. It shows the dependencies of all modules.
